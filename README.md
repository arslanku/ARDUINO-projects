# Открытый репозиторий Arduino/ESP проектов

Добро пожаловать в открытый репозиторий реализованных проектов на платформе Arduino и ESP. Здесь собраны работающие прототипы и решения для широкого круга задач — от систем умного дома и автоматизации до уникальных сценических костюмов и вендинговых автоматов.

Всего за активное время работы было создано **45 технических проектов**. Некоторые из них были скрыты в рамках соглашений с заказчиками, но большая часть представлена в этом репозитории для изучения, доработки и вдохновения.

## 🧭 Основные направления разработок

Проекты в репозитории охватывают следующие области:

*   **Промышленная автоматизация и АСУ:** Управление моторами, реле, сервоприводами, шаговыми двигателями, контроль времени работы оборудования и сотрудников.
*   **Умный дом и интернет вещей (IoT):** Интеграция с Яндекс Алисой, системы сигнализаций, удаленный контроль обогрева загородных домов, сбор данных с датчиков (температура, давление, влажность и др.).
*   **Беспроводные технологии:** Дистанционное управление с помощью радиомодулей 433 МГц, 2.4 ГГц, использование GSM-связи.
*   **Свет и спецэффекты:** Управление светодиодными лентами (включая адресные), разработка источников света, создание программируемых костюмов для выступлений артистов.
*   **Вендинг и платежные системы:** Опыт работы с купюроприемниками (например, CashCode) и вендинговыми автоматами.
*   **Прикладное ПО:** Разработка программ для Windows с функцией управления платой через компьютер.
*   **Образовательные и научные проекты:** Исходные коды, которые использовались для дипломных и научно-исследовательских работ.
*   **Модификация существующих решений:** Адаптация и доработка open-source проектов под конкретные требования заказчиков.

## 🚀 Как использовать

1.  **Клонируйте репозиторий:**
    ```bash
    git clone https://github.com/arslanku/ARDUINO-projects.git
    ```
2.  **Откройте интересующую вас папку** с номером проекта.
3.  Загрузите скетч (файл `.ino`) в **Arduino IDE**.
4.  Подключите плату согласно схеме (если схема не приложена, она может быть восстановлена по коду или описана в комментариях).
5.  Адаптируйте код под свои задачи и периферию.
